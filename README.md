# JuHub（聚汇）

> 聚会工具箱：分账、随机分组、投票决策……一个 PWA 装下朋友聚会需要的所有小工具。

前身是分账工具 Farely，2026-06 转型为工具箱。完整设计文档见 [DESIGN.md](DESIGN.md)。

## 为什么是工具箱

分账解决的是「聚会结束时」的问题；分组、投票解决的是「聚会进行中」的问题——**同一拨人、同一台手机、同一个场景，工具天然该长在一起**。

## 产品原则

这些原则是 Farely 时代验证过的，转型后不变：

1. **无登录、无后端、打开即用**——所有数据都在本机。
2. **链接即分享**——数据压缩编码进 URL，发个链接就是分享/备份；已发出去的旧链接永久兼容。
3. **手机优先的 PWA**，离线可用，可安装到主屏。
4. **一个人操作、全员受益**——记账员模式、传手机投票模式，不要求每个人都装应用。

## 整合的灵魂：名字池

如果每个工具都要重新输一遍名字，那 JuHub 只是个书签收藏夹。**让「这拨人」在工具间流动起来，才是工具箱成立的理由**——任何工具录入的名字都会进入共享名字池，其他工具输名字时点一下即填入。一晚上玩三个工具，名字只打一遍。

## 工具

| 工具 | 干什么 |
|---|---|
| 💸 分账 | 记账 AA、多币种、一键算出最少转账方案，链接分享/备份 |
| 🎲 随机分组 | 抽签分队，支持「必须同组」「必须分开」约束（情侣别拆 / 卧底别凑） |
| 🗳️ 投票决策 | 传手机投票（可匿名、平票随机决胜），或转盘随机定 |

投票坚持无后端，所以是**传手机模型**（本地诚实方案）；远程投票需要后端聚合，列入 V2（Supabase）。

## 加一个新工具有多便宜

`src/tools/` 下建一个目录 + 注册表（`src/tools/registry.tsx`）加一行，hub 自动出卡片；`React.lazy` 代码分割保证首屏不变重。候选池：顺序随机器、骰子、数字炸弹、真心话大冒险题库……

## 开发

```bash
npm install
npm run dev      # 开发服务器
npm test         # vitest
npm run build    # tsc + vite build
```

```
src/
  shared/        i18n、主题、弹层、头像、剪贴板、URL 载荷编解码、名字池
  tools/
    registry.tsx 工具注册表
    split/       分账
    teams/       随机分组
    vote/        投票决策
  Hub.tsx        工具中心首页
  router.ts      手写 hash 路由（零依赖）
```
